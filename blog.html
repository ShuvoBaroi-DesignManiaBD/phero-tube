Discuss the scope of var, let, and const

Scope determines the accessibility (visibility) of variables. Javascript has 3 types of scope: Block scope, Function scope & Global scope.

Var: Var has function scope, making it accessible throughout the function where it's declared or globally if declared outside any function. It's hoisted, allowing access before declaration.

Let: Let has block scope, limiting its access to the block (ex. loops or conditionals) where it's defined. It's not hoisted and can be reassigned within the same scope.

Const: Const also has block scope and cannot be reassigned after initial assignment, making it suitable for constants. Like `let`, it's not hoisted.

Tell us the use cases of null and undefined

Use Cases for NULL: Null serves several purposes. Firstly, it's used to indicate an absence of value, showing that a variable or property is intentionally empty or uninitialized. Secondly, it's handy for resetting variables, allowing you to clear the previous data while retaining the variable. Additionally, null can be employed as a placeholder value in data structures and algorithms, signifying that a value will be assigned or computed later. Lastly, in error handling, null can be returned by functions to signal errors or an inability to produce a valid result, providing a clear indicator of issues.

Use Cases for UNDEFINED:

  1. Implicit Absence of Value: undefined is automatically assigned to variables that exist but have not been given a value, indicating their absence of a meaningful value.

  2. Function Parameters: When calling a function with fewer arguments than expected, missing parameters are set to undefined, useful for conditional logic or default values.

  3. Property Access: Attempting to access a non-existent object property results in undefined, helping handle missing or optional properties.

  4. Implicit Return: Functions lacking a return statement or returning no value implicitly return undefined, often used in certain programming patterns.

What do you mean by REST API?

API means Application Programming Interface whethere REST API(Representational State Transfer Application Programming Interface) is a framework for designing web services with specific architectural principles. RESTful APIs are commonly used in web development to enable communication between software applications.

REST APIs are stateless, meaning each client-server interaction includes all necessary information. They employ a resource-based approach, with entities represented as resources accessed via standard HTTP methods.

In summary, RESTful APIs simplify communication between software components, promoting scalability and ease of use in web applications and services.
